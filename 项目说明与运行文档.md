# 项目说明与运行文档

## 1. 项目用途

本项目把微信或 QQ 聊天记录导出为统一格式的 CSV，然后对双人聊天做统计分析，生成本地 HTML 报告。

当前流程分成三步：

1. 从微信或 QQ 导出聊天记录到 CSV
2. 如有需要，把多个 CSV 合并成一份分析输入
3. 运行 `main.py` 生成图表、分析输入文件、人格结果 JSON 和 HTML 报告

## 2. 目录与文件作用

### 根目录脚本

- `main.py`
  读取标准化 CSV，输出统计图表、人格分析输入文件、自动生成的人格结果和 `report.html`。

- `export_contact_windows_v4.py`
  从已经解密的 Windows 微信 4 `db_storage` 目录导出单个联系人聊天记录。

- `export_qq_nt_c2c.py`
  从 Windows QQNT `nt_db` 目录导出私聊聊天记录。运行时需要提供 SQLCipher key。

- `merge_analysis_exports.py`
  合并多个导出 CSV，并按时间排序，生成新的分析输入文件。

### 核心包目录

- `chat_analyzer/analysis/loader.py`
  负责读取、清洗和标准化消息数据。

- `chat_analyzer/analysis/stats.py`
  负责统计计算。

- `chat_analyzer/analysis/sampler.py`
  负责抽样消息，生成人格分析输入。

- `chat_analyzer/analysis/personality.py`
  负责人格特征提取、结果归一化和自动人格生成。

- `chat_analyzer/analysis/emotion_periods.py`
  负责情绪波动与高压窗口分析。

- `chat_analyzer/analysis/topic_evolution.py`
  负责话题演化分析。

- `chat_analyzer/analysis/interaction_rhythm.py`
  负责互动节奏、回复延迟和开场统计。

- `chat_analyzer/analysis/role_inference.py`
  负责关系角色判断。

- `chat_analyzer/analysis/events.py`
  负责关键事件抽取。

- `chat_analyzer/analysis/identity_merge.py`
  负责多平台身份归并。

- `chat_analyzer/analysis/confidence.py`
  负责报告置信度计算。

- `chat_analyzer/reporting/visualizer.py`
  负责生成图表。

- `chat_analyzer/reporting/html_report.py`
  负责生成 HTML 报告。

### 工具目录

- `tools/export_contact_windows_v4.py`
  微信导出脚本主体，根目录同名文件是兼容入口。

- `tools/export_qq_nt_c2c.py`
  QQ 导出脚本主体，根目录同名文件是兼容入口。

- `tools/merge_analysis_exports.py`
  CSV 合并脚本主体，根目录同名文件是兼容入口。

- `tools/qqnt_element_pb2.py`
  QQNT 消息元素解析依赖。

- `chat_analyzer/utils/console.py`
  Windows 控制台编码兼容处理。

### 运行期目录

- `exports/`
  建议存放导出的 CSV 和对应的 `.meta.json`。这是运行产物，不建议提交到仓库。

- `outputs/`
  建议存放分析结果，例如 `report.html`、`charts/`、`personality_input.json`、`personality_result_generated.json`。这也是运行产物。

- `.venv/`
  本地 Python 虚拟环境目录。

## 3. 环境准备

建议使用 Python 3.10+。

```powershell
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

## 4. 微信导出命令

说明：当前微信导出脚本要求你已经拿到“解密后的 `db_storage` 目录”。项目本身不负责解密微信数据库。

### 列出联系人

```powershell
.\.venv\Scripts\python.exe .\export_contact_windows_v4.py `
  --db-dir "<DECRYPTED_WECHAT_DB_DIR>" `
  --list-contacts
```

### 按联系人名称导出

```powershell
.\.venv\Scripts\python.exe .\export_contact_windows_v4.py `
  --db-dir "<DECRYPTED_WECHAT_DB_DIR>" `
  --contact "联系人A" `
  --output ".\exports\wechat\contact_a.csv"
```

### 按微信 ID 导出

```powershell
.\.venv\Scripts\python.exe .\export_contact_windows_v4.py `
  --db-dir "<DECRYPTED_WECHAT_DB_DIR>" `
  --contact-id "wxid_example" `
  --output ".\exports\wechat\contact_a.csv"
```

导出完成后，通常会得到：

- `contact_a.csv`
- `contact_a.meta.json`

## 5. QQ 导出命令

说明：QQ 导出脚本要求你已经拿到 `nt_db` 目录和 SQLCipher key。项目本身不负责提取 QQ key。

### 列出私聊联系人

```powershell
.\.venv\Scripts\python.exe .\export_qq_nt_c2c.py `
  --db-dir "<QQ_NT_DB_DIR>" `
  --key "<QQ_NT_KEY>" `
  --list-contacts
```

### 按联系人导出

```powershell
.\.venv\Scripts\python.exe .\export_qq_nt_c2c.py `
  --db-dir "<QQ_NT_DB_DIR>" `
  --key "<QQ_NT_KEY>" `
  --contact "联系人B" `
  --output-dir ".\exports\qq\c2c"
```

导出完成后，输出目录里通常会有：

- `qq_<contact>.csv`
- `qq_<contact>.meta.json`
- `qq_c2c_contacts.json`

## 6. 合并多个 CSV

如果你想把微信和 QQ 的聊天一起分析，可以先合并 CSV：

```powershell
.\.venv\Scripts\python.exe .\merge_analysis_exports.py `
  ".\exports\wechat\contact_a.csv" `
  ".\exports\qq\c2c\qq_contact_b.csv" `
  --output ".\exports\merged\merged_chat.csv"
```

如果需要，也可以显式指定合并后的展示名称：

```powershell
.\.venv\Scripts\python.exe .\merge_analysis_exports.py `
  ".\exports\wechat\contact_a.csv" `
  ".\exports\qq\c2c\qq_contact_b.csv" `
  --output ".\exports\merged\merged_chat.csv" `
  --partner-name "合并对象"
```

## 7. 运行分析程序

### 直接生成完整分析

默认运行一次就会生成：

- 图表目录 `charts/`
- `personality_input.json`
- `partner_input.json`（如果有对方文本消息）
- `personality_result_generated.json`
- `partner_result_generated.json`（如果有对方文本消息）
- `advanced_insights.json`
- `report.html`

命令：

```powershell
.\.venv\Scripts\python.exe .\main.py `
  ".\exports\merged\merged_chat.csv" `
  --output ".\outputs\merged\merged_chat"
```

如果只分析单个平台的单份 CSV，也可以直接传那份文件：

```powershell
.\.venv\Scripts\python.exe .\main.py `
  ".\exports\wechat\contact_a.csv" `
  --output ".\outputs\wechat\contact_a"
```

### 用外部人格结果覆盖自动结果

如果你已经准备好了外部模型生成的人格分析结果 JSON，也可以再次运行，用它覆盖自动生成结果：

```powershell
.\.venv\Scripts\python.exe .\main.py `
  ".\exports\merged\merged_chat.csv" `
  --output ".\outputs\merged\merged_chat" `
  --personality-result ".\outputs\merged\merged_chat\personality_result.json" `
  --partner-personality-result ".\outputs\merged\merged_chat\partner_result.json"
```

完整报告主要输出：

- `report.html`
- `report.css`
- `charts/`
- `personality_result_generated.json`
- `partner_result_generated.json`
- `advanced_insights.json`
- `personality_raw.json`
